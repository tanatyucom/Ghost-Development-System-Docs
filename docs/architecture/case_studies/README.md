# Architecture Case Studies

## Purpose

This folder preserves the field evidence behind accepted GDS architecture
principles.

Case studies explain why a principle exists, what real project pressure shaped
it, which decisions were made, and what future Ghost projects can reuse.

## Case Studies

- `gameghost_repository_cleanup.md`: initial case study for Progressive
  Architecture Adoption and Architecture Lifetime Assessment, based on the
  GameGhost Repository Cleanup sequence.

## Relationship To Design Principles

Design principles are the long-term architectural guidance.

Case studies are supporting evidence. They should not introduce new mandatory
architecture, runtime behavior, quality gates, or implementation scope unless a
separate Q promotes that change.
