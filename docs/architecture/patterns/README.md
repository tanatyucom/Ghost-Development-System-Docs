# Architecture Pattern Candidates

## Purpose

This folder stores architecture pattern candidates that have enough evidence to
guide future review, but are not yet official GDS architecture patterns.

Pattern candidates preserve reusable architecture lessons before they are
promoted to rules, standards, SDK contracts, or official platform patterns.

## Pattern Candidates

- [Evidence-Driven Platform Promotion Pattern](evidence_driven_platform_promotion_pattern.md):
  Architecture Pattern Candidate for promoting reusable components from field
  project evidence through repository adoption, repeated validation, and Human
  Approval.

## Guardrails

- A pattern candidate is not an official pattern.
- A single successful reference implementation is evidence, not final proof.
- Promotion requires multiple-domain validation, Repository Adoption evidence,
  and Human Approval.
- Pattern registration does not authorize implementation, refactoring,
  migration, SDK extraction, commit, push, tag, or release work.
