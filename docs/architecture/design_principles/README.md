# Design Principles

## Purpose

This folder contains accepted GDS design principles.

Design principles guide architecture judgment across Ghost projects. They are
more stable than Q discussion records, but they do not implement runtime
behavior, scanner logic, quality gates, or repository mutation by themselves.

## Principles

- `progressive_architecture_adoption.md`: principle for deciding where
  engineering investment should be made during repository evolution, including
  Progressive Architecture Adoption, Architecture Lifetime Assessment,
  Repository Readiness Assessment, and Evidence-based Non-Implementation
  Decision.

## Relationship To Q Documents

Q documents preserve proposal, discussion, review, and approval history.

Design principles are the long-term source of truth for future project work.
When a Q is promoted into this folder, future Ghost projects should reference
the design principle first and use the original Q as traceability evidence.
