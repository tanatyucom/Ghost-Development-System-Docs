# Title

Memory Candidate (MC) Operation Guide v1.0

---

# Background

Memory Candidate (MC) has been officially adopted into GDS Governance.

The next step is to standardize daily operation so that every AI and every repository uses MC consistently.

This document defines operational guidance rather than architecture.

---

# Objectives

Establish a common operational procedure for:

- Creating MCs
- Reviewing MCs
- Promoting MCs
- Closing MCs

Reduce knowledge loss while avoiding unnecessary MC creation.

---

# When to Create a Memory Candidate

Create an MC when any of the following applies:

- Valuable architectural idea
- Workflow improvement
- Rule proposal
- Template proposal
- Naming proposal with long-term value
- Repository improvement
- Cross-project reusable knowledge
- User explicitly requests preservation
- Lost Context Risk is HIGH

---

# When NOT to Create a Memory Candidate

Do not create an MC for:

- Simple factual answers
- Temporary discussion with no long-term value
- Duplicate ideas already registered
- Completed Repository knowledge
- Small wording changes
- Casual conversation

---

# Lost Context Risk

Review before ending the session.

Question:

"If this conversation ends now, is valuable knowledge likely to be lost?"

If YES:

→ Create an MC before ending the session.

---

# Daily Workflow

Conversation
↓
Potential knowledge identified
↓
MC Created (if needed)
↓
Continue discussion
↓
Completion Review
↓
Memory Candidate Review
↓
Promotion decision
↓
Repository / Q / Memory
↓
Close MC

---

# Promotion Guidelines

Possible destinations:

- Memory
- Q
- Repository Draft
- Rule
- Workflow
- ADR
- Future Candidate

Repository remains the canonical destination.

---

# Deferred Review

Deferred items should be reviewed periodically.

Possible outcomes:

- Repository Registered
- Q Created
- Rejected
- Duplicate
- Expired

Deferred must never become permanent storage.

---

# Operational Principles

1. Capture before loss.
2. Repository is the Single Source of Truth.
3. MC is temporary.
4. Promote intentionally.
5. Close every MC.
