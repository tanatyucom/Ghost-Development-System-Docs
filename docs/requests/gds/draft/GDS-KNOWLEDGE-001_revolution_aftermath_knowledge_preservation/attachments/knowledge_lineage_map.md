# Knowledge Lineage Map

## Purpose

Show how the Intent-Driven Workflow discovery should be recoverable from
repository artifacts without relying on chat memory.

## Lineage

```text
Conversation / Design Discussion
  -> Q-GDS-INTENT-001 Intent-Driven Workflow Foundation
  -> GDS-INTENT-001 Completion Report
  -> Q-GDS-KNOWLEDGE-001 Revolution Aftermath Knowledge Preservation
  -> Intent-Driven Workflow Discovery Issa Draft
  -> Knowledge Artifact Responsibility Map
  -> Knowledge Preservation Gate
  -> ADR-GDS-005 Proposed
  -> Future Child Qs
  -> Completion Reports
  -> Improvement Record / Rule / Workflow / Template candidates
```

## Reconstruction Questions

A future reader should be able to answer:

- Why Intent-Driven Workflow exists.
- Why natural language is not direct execution.
- Why `お願いします` needs Pending Action context.
- Why Human Approval remains mandatory for mutating actions.
- Why GDS assumes humans and AI forget.
- Which artifact preserves what changed, what was decided, and why.
