# Session Summary Standard

**Status:** Adopted
**Version:** 1.0

Generate a Session Summary when a session materially changes design, state,
scope, evidence, risk, or the next entry point.

Required sections:

- Session identity, goal, time, repository revisions, and source artifacts;
- Current State before and after;
- completed and incomplete work;
- Decision Summary and Decision History updates;
- concise Reasoning Summary: evidence, constraints, alternatives, rationale;
- Architecture Change Summary;
- Learning Summary;
- risks, assumptions, open questions, and blockers;
- validation and changed files;
- approval and Git-operation state;
- Next Session Entry Point and required reading order.

Do not include hidden chain-of-thought, secrets, unverifiable memory claims, or
long chat transcripts. Link to canonical evidence instead of duplicating it.
