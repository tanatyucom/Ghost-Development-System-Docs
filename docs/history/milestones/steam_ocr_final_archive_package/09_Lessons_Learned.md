# 09 Lessons Learned

- 単一サンプルで最適化しない。
- Metricsだけで成功判断しない。
- Intermediate Artifactを見る。
- Metricsと現物が矛盾したら採用停止する。
- Trace Before Tune。
- First Broken Stepを探す。
- Candidate Generationが壊れていればScoringでは直らない。
- Negative Resultを残す。
- GUIはHuman Review疲労を減らすInfrastructureである。
- Research Qは実装Qと分ける。
- AI assists, Human decides。
- Knowledgeは保存、分類、昇格、物語化して初めて再利用できる。
