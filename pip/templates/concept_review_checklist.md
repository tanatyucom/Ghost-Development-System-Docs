# Concept Review Checklist

Use this checklist when reviewing a concept under `pip/concepts/`.

## Scope Check

- [ ] Concept ID follows `CONCEPT-YYYY-NNN`.
- [ ] Concept Index was checked before assigning a new ID.
- [ ] The concept is not a request to implement runtime behavior by itself.
- [ ] The source Q, report, Roadmap2 note, or PIP section is linked.
- [ ] The concept is not already covered by an existing rule, workflow, glossary term, CASE, or template.
- [ ] The concept has a clear problem or knowledge gap.

## GDS Principle Check

- [ ] The concept does not conflict with Evidence First.
- [ ] The concept does not conflict with Human Approval Gate.
- [ ] The concept does not conflict with Knowledge Before Automation.
- [ ] The concept does not conflict with Artifact First.
- [ ] The concept does not conflict with Project First.
- [ ] The concept does not silently expand scope.

## Validation Check

- [ ] The concept can be tested in an actual project, Q, review, or documentation workflow.
- [ ] The expected benefit is clear.
- [ ] Required evidence is available or can be produced.
- [ ] Review result can be recorded in a Concept Status Change Report.

## Promotion Decision

- [ ] The concept is mature enough for Rule.
- [ ] The concept is better as Workflow.
- [ ] The concept is better as Best Practice.
- [ ] The concept is better as CASE.
- [ ] The concept is better as Rule Story.
- [ ] The concept is better as Evolution.
- [ ] The concept is better as Investigation.
- [ ] The concept is better as Template.
- [ ] The concept is better as Glossary.
- [ ] The concept is better as Architecture.
- [ ] The concept should stay Candidate.
- [ ] The concept should move to Under Review.
- [ ] The concept should move to Experiment.
- [ ] The concept should move to Validated.
- [ ] The concept should be Archived.

## Archive Check

- [ ] It duplicates existing knowledge.
- [ ] It has no reuse value outside one task.
- [ ] It lacks reliable evidence.
- [ ] It conflicts with GDS principles.
- [ ] It is a Future Candidate outside current scope.
- [ ] It was replaced by a clearer concept or stronger knowledge unit.

## Required Output

- [ ] Concept file updated.
- [ ] Concept Index updated.
- [ ] Concept Status Change Report created or updated.
- [ ] Related docs updated.
- [ ] Next action recorded.
