# PIP Evolutions

## Purpose

Evolution entries record how repeated cases changed the project workflow, rules, templates, or architecture direction.

## Expected Content

- Evolution ID
- Date
- Related cases
- Before state
- Trigger
- Change made
- New standard
- Remaining future candidates

## Use

Use this folder when one issue becomes a chain of reusable knowledge rather than a single isolated case.
