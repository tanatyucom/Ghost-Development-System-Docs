# PIP Rule Stories

## Purpose

Rule Stories explain why a rule exists by linking it to real cases and evidence.

They are shorter than a full rule document and more reusable than a single completion report.

## Expected Content

- Rule Story ID
- Title
- Related Case
- Related Rule
- Problem pattern
- Why the rule prevents recurrence
- Evidence links
- Promotion status

## Examples

- Trace Before Tune
- Human Visual Review
- Review Entry Point
- Audit Before Repair
- Migration First
