## Background

Roadmap2 generated many artifacts and required a clear first reference point
for later human and AI review.

## Problem

If a debug package has many files, reviewers may not know which artifact is
authoritative or where to begin.

## Proposed Idea

Define a Debug Reference: the primary artifact, report, contact sheet, table, or
entry point that reviewers should inspect first.

## Expected Benefit

This reduces review confusion and improves handoff quality.

## Validation Plan

Use when completion reports or review packages contain multiple artifacts.
