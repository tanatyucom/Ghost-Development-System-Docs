# GameGhost Repository Profile

## Production Repository

- Project: GameGhost
- Production Repository path: `C:\GrayGhostArchive`
- Repository type: production runtime repository

## GDS Documentation Repository

- Project Profile owner: Ghost Development System Docs
- Profile path: `project_profiles/gameghost/`
- GDS Docs path: `C:\GitHub\Ghost-Development-System-Docs`

## Backup / Reference Repository Policy

Backup, copied, downloaded, temporary, or reference-only GameGhost folders must
not be edited unless a Q explicitly names them as the working repository and
the user approves that scope.

When GameGhost is only the related repository for a GDS documentation task,
GameGhost is reference-only.

## Required Startup Checks

Before GameGhost implementation, review, repair, commit, tag, or release:

```text
1. Confirm Q Target Project is GameGhost.
2. Confirm Working Repository.
3. Run repository root validation.
4. Confirm production / backup / reference-only boundary.
5. Confirm Scope / Out of Scope.
6. Confirm dirty workspace and commit policy.
```

The expected Git root must match the Q Working Repository before edits begin.

## Edit Boundary

Do not edit GameGhost runtime files from a GDS documentation Q.

Do not copy, sync, or migrate files between GDS Docs and GameGhost without a
separate approved Q.
