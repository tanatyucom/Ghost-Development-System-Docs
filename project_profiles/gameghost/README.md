# GameGhost Project Profile

## Purpose

This profile records stable GameGhost-specific context for AI and human work.

Use it after reading GDS shared rules and before executing a GameGhost Q.

## Reading Order

```text
docs/ai_repository_index.md
  -> docs/rules/rules.md
  -> docs/workflow/README.md
  -> project_profiles/gameghost/README.md
  -> GameGhost Q file
  -> Startup Checklist
```

## Contains

- `repository.md`: repository locations and edit boundaries.
- `rules.md`: GameGhost-specific work rules.
- `workflow.md`: GameGhost-specific workflow expectations.
- `ai_context.md`: AI context for GameGhost work.
- `completion_policy.md`: completion report, verification, commit, and release
  expectations.

## Scope

This profile is documentation in GDS Docs. It does not edit GameGhost runtime
files and does not approve GameGhost implementation by itself.

GameGhost implementation still requires a scoped Q, Startup Checklist,
Repository Root Validation, and the relevant GDS rules.
